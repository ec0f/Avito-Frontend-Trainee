# Avito Frontend Trainee Assignment Spring 2026

Тестовое задание на позицию Frontend Trainee.

## О проекте

Приложение для работы с объявлениями:

- просмотр списка объявлений;
- фильтрация, поиск, сортировка и пагинация;
- просмотр карточки объявления;
- редактирование объявления;
- сохранение изменений через API;
- сохранение черновика в `localStorage`;
- AI-помощник для:
  - генерации / улучшения описания;
  - подсказки ориентировочной цены.

## Структура проекта

```text

```

### Запуск проекта
1. Клонирование репозиотрия

```text
git clone https://github.com/ec0f/Avito-Frontend-Trainee
```
2. Установка зависимостей клиента

```text
cd client
npm install
```
3. Установка зависимостей сервера

```text
cd server
npm install
```
4. Настройка переменных окружения

Сервер
- Создать файл .env в папке server:
```text
GIGACHAT_AUTH_KEY=your_auth_key_here
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat
PORT=8080
```

Клиент
- Создать файл .env в папке client:
```text
REACT_APP_AI_ENDPOINT=http://localhost:8080/ai
REACT_APP_API_URL=http://localhost:8080
```
5. Запуск сервера
- Из папки сервера
```text
npm start
```
6. Запуск клиента
- Из папки клиента
```text
npm start
```
7. Настройка LLM (GigaChat)
- Для работы AI-помощника используется GigaChat.
- Что нужно сделать:
1. Создать проект GigaChat API в личном кабинете.
2. Получить Authorization Key.
3. Вставить его в server/.env в переменную:
```text
GIGACHAT_AUTH_KEY=...
```
### Самостоятельно принятые решения
1. Клиент и сервер разделены на два независимых приложения

Решение: frontend и mock backend вынесены в разные папки (client и server) и запускаются отдельно.

Почему:
Это упрощает разработку, не смешивает зависимости и делает запуск более предсказуемым.

2. В ответ списка объявлений добавлен id

Решение: в GET /items был добавлен id для каждого элемента списка.

Почему:
Фронтенду нужен идентификатор, чтобы можно было корректно переходить со списка на страницу конкретного объявления.

3. Исправлен локальный запуск сервера

Решение: сервер был адаптирован для корректного запуска в локальной среде, включая Windows / PowerShell.

Почему:
Исходный шаблон сервера требовал небольших доработок для стабильного старта и корректной работы с фронтендом.

4. Добавлен корректный CORS

Решение: на сервере был настроен CORS для взаимодействия клиента (localhost:3000) и сервера (localhost:8080).

Почему:
Без этого браузер блокировал запросы между фронтендом и backend.

5. Черновик формы хранится в localStorage

Решение: состояние формы редактирования сохраняется локально до отправки на сервер.

Почему:
Это соответствует требованиям задания и улучшает пользовательский опыт, так как данные не теряются после обновления страницы.

6. GigaChat интегрирован через backend-proxy

Решение: вызов LLM вынесен на сервер, а не выполняется напрямую из браузера.

Почему:
Так безопаснее, потому что ключ не попадает в клиентский код, и проще поддерживать AI-интеграцию в дальнейшем.

7. Ответ AI отображается в popover рядом с кнопкой

Решение: результат работы AI показывается в отдельном popover с действиями «Применить» и «Закрыть».

Почему:
Такой формат ближе к ожидаемому UX по макету и позволяет пользователю явно контролировать применение предложенного значения.

8. Логика состояния разделена на несколько MobX store

Решение: состояние приложения разделено на отдельные store:

список объявлений;
просмотр объявления;
редактирование объявления.

Почему:
Это делает код проще для чтения, поддержки и дальнейшего расширения.
